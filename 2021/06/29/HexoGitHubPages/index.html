使用Hexo在GitHub Pages上搭建自己的个人博客

0. 搭建个人博客意义

就算是像我一样的小白，也知道在学习新知识之后对于内容的归纳总结输出练习的重要性，包括但不限于笔记、心得、一小段Demo等等。这些都可以发到个人的博客上面，方便自己查阅复习。反过来说，自己踩的坑都可以记录下来，可以避免以后反复踩同一个坑。
稍微功利一点，很多从业者都说，与工作相关的博客在面试的时候是一个加分项。至少写博客没有什么坏处。

当然，我是小白，没法像大神那样写出什么很有意义或者很厉害的东西。但是我没有理由不写，就像不是专业厨师就不配做饭拍照发朋友圈了嘛？那么在写博客之前，先搭建自己的个人博客吧！

当然，搭建博客的时候，肯定~会出现问题的。这不严重，大不了删了从头再来一遍罢了。

那在这篇文章是从零开始一步一步，就像标题那样，（在Windows系统中）用Hexo在GitHub Page上部署自己的个人博客。里面也写了一些我遇到的问题以及我找到的解决方法，以及某些我找到的偷懒办法。
当然在别的操作系统用别的（静态）网站框架在别的服务器上部署的步骤应该可以类比吧。

注：以下内容默认阅读的人了解命令提示符（按Win+R，输入cmd并运行）中的基本操作。

1. 部署环境

安装Hexo之前，首先需要安装Node.js和Git。以及为了之后的许多操作，甚至之后的编辑文章，我个人推荐再安装一个支持多语言的IDE。

Node.js

首先检查自己电脑中是否安装了Node.js。打开CMD，输入node -v。此条命令是检查电脑中Node.js的版本。如果返回了版本号，则说明已安装Node.js。
若显示“错误”，或者版本号过低：

  • 下载官网上稳定版本的安装程序并安装
  • 确保安装过程中，勾选“Add to PATH”选项

Git

和之前同理，检查是否安装了Git。打开CMD，输入git -v。
若显示“错误”：

  • 官网下载页下载安装程序并安装

IDE（可选，但推荐）

我个人习惯是用VS Code。主要原因是好看。它可以通过安装第三方插件来支持几乎所有的语言（高亮语法结构），也就是说除了特定需求我不需要安装别的IDE了。

2. 安装Hexo并部署

  1. 将CMD路径调到想要安装的文件夹路径。个人建议新建一个文件夹，这样万一安装错了可以直接删掉重来。
  • （可选）用淘宝镜像源来安装后续内容。在CMD中输入npm install -g cnpm --registry=https://registry.npm.taobao.org并安装完成之后输入cnpm -v，如果显示版本号说明安装成功。之后所有带有npm的指令改为cnpm就可以从淘宝源上下载内容了。
  1. 在CMD中输入npm install -g hexo-cli。
  2. 安装完成后输入hexo -v，若显示版本号则说明安装成功。
  3. 输入hexo init <blog-folder-name>，这样会在现有的路径中创建一个<blog-folder-name>的文件夹并在里面安装。
  • 若提示安装不成功ERROR（常见），则运行
  1. 运行hexo s，默认会在本地4000端口中生成一个网站的访问链接。打开浏览器输入http://localhost:4000，如果成功的话就能见到一个建立起来的网站。
    注：理论上hexo s命令会监视文件变动并自动更新，所以可以之后用作预览生成的网站。命令其他选项参见官方文档。

3. 写作以及页面属性

打开一个新的CMD窗口（这样本地端口的网站页面会实时刷新），并把工作目录调至<path>\<blog-folder-name>。此时如果安装了IDE，推荐直接在IDE界面以及内嵌命令提示符中操作。

  1. 生成文章
    输入hexo n "<post-title>"（注：由于CMD格式引号必须是双引号）。系统会在<path>\<blog-folder-name>\source\_posts中生成一个Markdown（.md）文件。可以打开这个文件并编辑博文内容（如果没有IDE就只能在记事本或者CMD窗口中编辑了）。可以输入以下内容来测试效果。 此时刷新本地页面，上面会有一篇新的博文。如果没有则在CMD中输入

  1. 生成草稿，发布
    输入hexo new draft "<draft-title>"。系统会在<path>\<blog-folder-name>\source\_drafts中生成草稿。草稿默认不会被渲染，也就是说最后生成静态网站的时候不会显示草稿。
    发布草稿可以通过输入$ hexo publish <filename>。但是我个人会直接在IDE中将草稿文件直接拖入_posts文件夹。

  2. 生成页面及其他
    输入hexo new page "<page-title>"。系统会在<path>\<blog-folder-name>\source中生成“page title”的文件夹，里面有一个名为“index.md”的文件。
    hexo new是根据<path>\<blog-folder-name>\scaffolds内的模板来生成文件的，可以自定义添加编辑模板。

  3. 页面属性/Front-Matter
    在每篇.md的博文文件中最上方会有一块被三条横杠分隔开来的区域，这就是Front matter。里面具体能加入的参数格式及其描述可以在官方文档页面中查看。

4. 配置文件

配置文件就是名为_config.yml的文件。一开始并不需要更改很多参数，是往后个性化时会用得到。配置文件分为网页配置文件，在〈blog-folder-name〉文件夹下，和主题配置文件，在<path>\<blog folder name>\themes\<theme-name>下（主题的安装在之后的小节中讲）。
网站配置文件中所有参数格式及其描述可以在官方文档页面中查看，这里只说几个最基本，一开始就可以更改的参数：

参数 描述 说白话
title 网站标题 - 浏览器标签上显示的内容
subtitle 网站副标题 - 某些主题中打开主页的内容第一行字
description 网站描述 - 某些主题中打开主页怼在脸上的大字
keywords 网站关键字 - 搜索引擎在搜索的时候首先找到的内容
author 作者名字 - 会在博客博文各个地方都提醒人这都是谁写的

5. 在GitHub Pages上部署

  1. 既然想在GitHub上部署，那么首先需要有自己的GitHub账号吧（废话）。

  2. 新建一个“〈username〉.github.io”的库，并设置为public，往后便可以直接通过这个域名样式的库名浏览这个博客了。（当然，可以是别的名称，但是首先还是需要.io作为库名结尾，而且这样往后就需要访问〈username〉\〈respository-name〉.io来访问博客）

  3. 因为GitHub从2021年中（现在）开始便不支持远端（Git和API Operation）通过用户名密码来访问编辑储存库了，所以需要生成一个个人访问令牌（Token）。官方页面有详细步骤和注意事项，这里简单说明一下。
    GitHub页面右上角个人资料照片→设置（Settings）→左侧边栏开发者设置（Developer settings）→左侧边栏个人访问令牌（Personal access tokens）→右侧生成新令牌（Generate new token）。然后给令牌一个名称以及勾选权限（目前为止我只勾选了repo权限没有遇到任何问题），然后生成令牌并保存到本地（因为离开页面以后就再也看不到令牌了）。

6. 主题

Hexo官网和GitHub上有许多免费的主题，可以选择一个自己喜欢的然后将自己的博客个性化成自己想要的样子。

  1. 下载主题。在cmd中输入git clone <theme-url> themes/<theme-name>，其中Theme URL应该是一个类似于githhub.com/<some-username>/hexo-theme-<theme-name>.git的地址。确认在<path>/themes文件夹下多出来了下载的主题文件夹。
  2. 打开〈blog-folder-name〉文件夹下的_config.xml，将“theme”的值修改为〈theme-name〉。
  3. 打开<path>/themes/<theme-name>中的_config.xml，将里面的属性按自己需要设置。
  4. 这之后重新生成一遍页面应该就能在本地端口看到跟换的主题了。
